Reading the sheet (above) needs nothing extra. To let the Compose form add rows directly to your sheet: open your sheet → Extensions → Apps Script → delete any starter code → paste the script below (after picking your own secret key, see next field) → Deploy → New deployment → type "Web app" → Execute as: Me → Who has access: Anyone → Deploy → copy the URL ending in /exec and paste it here.
This is what makes posting authorized instead of open to anyone who finds your URL. Pre-filled to admin@buzamedia to match the SECRET line in the script below — change both together if you ever want a different phrase. Requests without the right key are rejected before they touch your sheet.
This script never asks anyone visiting your site to sign in with Google, and never shows a "Google hasn't verified this app" warning — that screen only ever appears to you, once, the moment you personally authorize the deployment. Because it's "Execute as: Me," it always runs with your permissions, quietly, for every request. Note: browsers can't read an Apps Script Web App's response due to CORS, so the site can't confirm success/failure after sending — double-check the sheet itself, or open Apps Script → Executions to see real error messages (like a rejected secret key). Also remember: if you edit the script again later, you must create another new deployment — saving alone does not update the live URL.
How it works: the sheet must be shared as "Anyone with the link — Viewer". Once connected, Buza Media reads your sheet's rows as articles every time the site loads (cached briefly in this browser for speed). Editing a post here in the admin panel only changes your local copy — to update the live site for everyone, edit the row in Google Sheets and sync again.
